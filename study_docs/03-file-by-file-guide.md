# 파일별 역할 설명

## `main.c`
역할:
- 프로그램의 시작점이다.

핵심:
- `main()` 함수 하나만 있다.
- 실제 일은 거의 하지 않고 `run_repl()`을 호출한다.

왜 이렇게 했는가:
- `main.c`를 단순하게 유지하기 위해서다.
- 나중에 입력 방식이 바뀌어도 `repl.c`만 수정하면 되게 하려는 구조다.

## `repl.c`
역할:
- 프롬프트를 출력한다.
- 사용자 입력을 받는다.
- 여러 줄 입력을 누적한다.
- `exit`, `quit`를 처리한다.
- 파서와 실행기를 호출한다.

이 파일은 프로그램의 "대화형 콘솔" 역할을 한다.

## `parser.c`
역할:
- 사용자가 입력한 문자열을 해석한다.

예:
- `INSERT`인지
- `SELECT`인지
- 테이블 이름이 무엇인지
- 값이 몇 개인지
- `WHERE` 조건이 있는지

즉 이 파일은 "문자열을 구조체로 바꾸는 곳"이다.

## `executor.c`
역할:
- 파서가 만든 결과를 보고 무엇을 할지 결정한다.

예:
- `INSERT`면 `storage.c`를 이용해 파일에 추가
- `SELECT`면 `storage.c`로 읽고 `printer.c`로 출력

즉 "해석 결과를 실제 행동으로 연결하는 중간 관리자"라고 생각하면 된다.

## `storage.c`
역할:
- `data/users.csv` 파일을 읽고 쓴다.

여기서 하는 일:
- CSV 한 줄 추가
- CSV 전체 읽기
- CSV 한 줄을 컬럼으로 나누기
- 조건에 맞는 행인지 확인하기

## `printer.c`
역할:
- 화면 출력 형식을 담당한다.

예:
- 컬럼 헤더 출력
- 한 행 출력
- `Inserted 1 row`
- `No rows found`

## `utils.c`
역할:
- 문자열을 다루는 작은 보조 함수들을 모아둔다.

예:
- 줄바꿈 제거
- 앞뒤 공백 제거
- 숫자인지 검사
- 빈 문자열인지 검사

## `types.h`
역할:
- 여러 파일이 공통으로 쓰는 자료형을 정의한다.

예:
- `CommandType`
- `ParseStatus`
- `ExecStatus`
- `Command`
- `InsertCommand`
- `SelectCommand`

이 파일은 여러 `.c` 파일이 같은 약속으로 데이터를 주고받게 해준다.
