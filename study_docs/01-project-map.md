# 프로젝트 한눈에 보기

## 이 프로젝트는 무엇인가
이 프로젝트는 C 언어로 만든 작은 `MiniSQL` 프로그램이다.
사용자는 터미널에서 `INSERT`, `SELECT` 같은 문장을 입력하고, 프로그램은 그 문장을 해석해서 `data/users.csv` 파일을 읽거나 수정한다.

아주 짧게 말하면:

`사용자 입력 -> 문장 해석 -> 실행 -> CSV 파일 읽기/쓰기 -> 결과 출력`

## 가장 중요한 흐름
예를 들어 사용자가 이런 문장을 입력했다고 하자.

```sql
SELECT * FROM users WHERE id = 1;
```

프로그램은 대략 이렇게 움직인다.

1. 사용자의 입력을 받는다.
2. 이 문장이 `SELECT`라는 것을 알아낸다.
3. `users` 테이블을 조회해야 한다는 것을 알아낸다.
4. 조건이 `id = 1`이라는 것을 알아낸다.
5. `data/users.csv` 파일을 읽는다.
6. 조건에 맞는 행만 찾는다.
7. 결과를 화면에 출력한다.

## 이 프로젝트를 구성하는 큰 부품
- `main.c`
  프로그램 시작점
- `repl.c`
  사용자 입력을 계속 받는 부분
- `parser.c`
  문자열을 해석하는 부분
- `executor.c`
  해석된 결과를 실제 동작으로 연결하는 부분
- `storage.c`
  CSV 파일을 읽고 쓰는 부분
- `printer.c`
  화면에 출력하는 부분
- `utils.c`
  문자열 처리에 필요한 보조 함수들

## 왜 파일을 나눴는가
처음 보면 "한 파일에 다 넣으면 안 되나?" 싶을 수 있다.
하지만 파일을 나누면 역할이 분명해진다.

예:
- 입력 받는 로직은 `repl.c`
- SQL 해석은 `parser.c`
- CSV 처리는 `storage.c`

즉, 각각의 파일이 "한 가지 일"에 집중하도록 나눈 것이다.
