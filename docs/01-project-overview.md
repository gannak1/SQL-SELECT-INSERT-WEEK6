# 프로젝트 개요와 범위

## 프로젝트 목표
이 프로젝트의 목표는 C 언어로 간단한 SQL 처리기를 구현하는 것이다.
프로그램은 `MiniSQL> ` 프롬프트를 표시하는 인터랙티브 CLI 환경에서 과제용 `MiniSQL` 문장을 반복 입력받아 해석한 뒤 파일 기반 저장소에 대해 동작을 수행한다.

핵심 흐름은 다음과 같다.

입력(인터랙티브 CLI MiniSQL 문자열) -> 파싱 -> 실행 -> 파일 저장/조회

## 과제에서 중요한 점
- MiniSQL 문장을 읽고 해석하는 처리기를 만든다.
- `INSERT`, `SELECT`를 최소 지원한다.
- DB 대신 파일을 저장소로 사용한다.
- CLI 프로그램 형태로 동작한다.

## 1차 구현 범위
### 포함
- 인터랙티브 CLI 입력 처리
- `INSERT`
- `SELECT`
- 선택적 `WHERE column = value`
- 파일 기반 저장

### 제외
- `CREATE TABLE`
- `UPDATE`
- `DELETE`
- `JOIN`
- `ORDER BY`
- `GROUP BY`
- 복합 조건 `AND`, `OR`

## 기본 가정
- schema와 table은 이미 존재한다고 가정한다.
- 테이블 생성 기능은 구현하지 않는다.
- 구현 초반에는 최소 범위를 먼저 완성한다.
- 1차 구현에서는 임의의 여러 테이블을 지원하지 않고, 하드코딩된 `users` 테이블 하나만 사용한다.

## 입력 방식
이 프로젝트는 SQL 파일을 읽는 방식이 아니라, 실행 후 `MiniSQL> ` 프롬프트를 표시하고 사용자의 MiniSQL 입력을 반복해서 받는 방식을 사용한다.

여기서 `MiniSQL`은 표준 SQL 전체가 아니라, 이번 과제에서 지원하는 제한된 문법만 포함하는 입력 형식이다.

예:
```text
$ ./sql_processor
MiniSQL> INSERT INTO users VALUES (
...> "hong13", "Hong", 26,
...> "010-1313-1414", "hong@example.com"
...> );
MiniSQL> SELECT * FROM users;
MiniSQL> exit
```

## 인터랙티브 입력 규칙
- 프로그램은 시작 후 `MiniSQL> ` 프롬프트를 출력한다.
- 사용자는 MiniSQL 문장을 한 줄씩 입력하고, 세미콜론이 나올 때까지 여러 줄로 이어서 입력할 수 있다.
- 문장 누적 중에는 `...> ` 프롬프트를 출력한다.
- 문장 끝에는 세미콜론 `;`을 붙인다.
- `exit` 또는 `quit` 입력 시 프로그램을 종료한다.
- 빈 줄은 무시하고 다시 입력을 받는다.

## 중요한 설계 질문
이 프로젝트는 아래 질문에 대한 답을 먼저 정리해야 한다.

1. MiniSQL 문장을 어떤 형태로 구조화할 것인가
2. 테이블 구조 정보는 어디에 둘 것인가
3. 실제 데이터는 어떤 파일 포맷으로 저장할 것인가
4. 출력은 어떤 형태로 보여줄 것인가

## 1차 고정 테이블
초기 구현 단순화를 위해 회원가입 시 생성되는 사용자 테이블이 이미 존재한다고 가정한다.
즉, 1차 구현에서는 `users` 테이블만 지원한다.

세부 컬럼 구조는 [하드코딩 테이블 정의](06-users-table.md) 문서에서 관리한다.
